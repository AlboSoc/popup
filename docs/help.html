Popup Operator Help

The app edits strip-based kirigami designs of the form { widths, strips }. Each strip alternates horizontal and vertical travel lengths, and every strip must satisfy the shared endpoint invariant hsum(strip) === vsum(strip) === H.

Regression checks now live outside the UI. Run npm test from app_v2 to execute the operator test harness.

Operators

max(a, b)
Union of two compatible staircase profiles.
min(a, b)
Intersection of two compatible staircase profiles.
add(a, b)
Adds heights stripwise and clamps to H.
sub(a, b)
Monotone envelope of the stripwise difference, clamped at 0.
blend(a, b, t)
Interpolates between compatible profiles with t in [0, 1].
clamp(a, lo, hi)
Constrains a design between lower and upper envelopes.
mirror(a)
Reverses strip order and strip widths.
pad(a, start, end)
Adds symmetric staircase clearance at the beginning and end of every strip.
repeat(a, n)
Tiles a design across the page width n times.
concat(a, b, ...)
Places compatible designs side-by-side, then normalizes any overlaps into a single positive-width design.
offset(a, before, after)
Adds plain folded margin strips before and after a design. Widths may be negative, which acts as overlap sugar before final normalization.
Paper(length, width, height)
Creates a plain folded sheet whose 3D domain is [0, width] x [0, height] x [0, length] in (x, y, z). So width is the established horizontal paper width from the core strip-width data, height is the upward 3D y limit for dropped boxes, and length is the paper extent along 3D z. In the 2D paper coordinates this same length is the popup depth parameter H, so the sheet runs from t = 0 to t = 2 * length.
dropBox(model, length, width, height, position)
Splits the paper model as needed and adds the effect of a physical box dropped onto the popup. The box spans length along 3D z, width along 3D x, and height along 3D y. It lands flush against the z = 0 plane, so every covered strip gets a vertical increment of height over the final length portion of that strip. Repeating the same drop stacks again, and if the current paper is too short or too low to accommodate the added height, the paper automatically grows. position places the box across the paper width using the shared horizontal x coordinate; a number means the left edge, so position = p covers x in [p, p + width].
dropBoxes(model, specs)
Applies many dropped boxes in sequence using Array.prototype.reduce-style composition. Each spec may be [length, width, height, position] or { length|l, width|w, height|h, position|p }.

Expression Examples

mirror(step)

pad(gate, 0.2, 0.35)

repeat(step, 3)

concat(step, mirror(step))

offset(step, [0.5, 0.5], 0.75)

concat(saw, offset(mirror(saw), -2))

let scene = Paper(2, 6, 1.3);
scene = dropBox(scene, 0.8, 1.2, 0.5, 1.2);
scene = dropBox(scene, 1.1, 1.6, 0.9, 3.0);
scene = dropBox(scene, 0.7, 1.0, 0.4, 4.8);
return scene;

dropBoxes(Paper(2, 6, 1.3), [
  [0.8, 1.2, 0.5, 1.2],
  [1.1, 1.6, 0.9, 3.0],
  { l: 0.7, w: 1.0, h: 0.4, p: 4.8 }
])

const variant = (() => {
  const out = clone(gate);
  out.strips[2] = [0.5, 1.1, 1.0, 0.25, 0.5, 0.65];
  return out;
})();
return clamp(blend(gate, variant, 0.4), min(gate, variant), max(gate, variant));

Tests

Run npm test from app_v2. The test harness loads the app under Node, then runs the dedicated operator regression suite from tests/operator-tests.js.

2D Pattern Legend

board edge, cut, valley fold, mountain fold.

The 2D pane now draws internal vertical cut segments only where adjacent strips differ in 3D state, so slits show up as actual openings instead of full-height strip separators.

3D Controls

Mouse drag rotates, the wheel zooms, right-drag pans, and the view cube or keys F B L R T D switch orthographic views. Esc resets to the drafting-style default view. With idle orbit enabled, the 3D view also slips into its screensaver-style tumble after 15 seconds without 3D interaction or after a large zoom change.
